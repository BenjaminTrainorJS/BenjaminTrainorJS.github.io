sprint two - technical: classes vs. ids

What are the best practices associated with using classes vs. ids?

22 April 2020

.classes

A class attribute `class="header"` is used as a hook, to create a link between the CSS selector `.header` in the stylesheet to the HTML element. The class attribute can be 'hooked' to multiple HTML elements. This is done to save writng, or copy and pasting code multiple times. The class attribute `class="header"` should be specific to the semantic HTML tag which is being 'hooked' to, but the attribute doesn't change the semantic meaning of the HTML structure. The class attribute is not unique, and can be used multiple times.

#ids

Unlike the CSS `class=""` attributes, the `id="" attributes are unique, and are only to be used once per HTML page. These ids are best used to provde a point of reference or hook for internal links within the same HTML document ie linking from the top to the bottom of the page. The id is used within the href="" attribute, for example using the id="footer" to hook into the < footer id="footer"> tag and then usign a link tag with the href="#footer" attribute.
